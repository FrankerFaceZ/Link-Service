normalizeURL(url, base, default_schemeopt) → {URL}

Source:
  • normalize-url.js, line 25

Normalize a URL.

This adds a scheme to URLs without a scheme, ensures that the scheme is either HTTP or HTTPS, removes search hashes, removes trailing dots from hostnames, and removes garbage query parameters such as utm_ tracking parameters to slim down links for better cache hit rates.

If no base URL is provided and the URL is a relative URL, a RelativeURLError will be thrown. If the scheme is not HTTP or HTTPS, an UnsupportedSchemeError will be thrown.

Parameters:
Name Type Attributes Default Description
url String | URL

The URL to normalize.
base URL

The base URL for processing relative URLs.
default_scheme String <optional>
 'http'

The default scheme to set if a URL has no scheme.
Returns:

The normalized URL

Type
URL

Properties:
Name Type Attributes Default Description
url String | URL

Automatic. The current URL of the request. May change after Resolver#transformURL.
original_url URL

Automatic. The original URL of the request.
request FetchResponse

Automatic. The fetch request object for this request. Not available within Resolver#transformURL.
referrer URL

The referrer of the request. May be changed within Resolver#transformURL to change the Referer header sent with the request.
fetch function <optional>

The version of fetch to use. Useful in case you want to wrap fetch with something to, for example, automatically renew a client credential token when necessary and add it to the request. The custom fetch implementation should wrap our custom fetch with timeout support or provide an equivilent API.
headers Object <optional>

An optional object of headers to send with the request. Has no effect if set outside Resolver#transformURL.
options Object <optional>

An optional object of options to send to fetch when performing the request. Has no effect is set outside Resolver#transformURL.
timeout Number <optional>

The number of miliseconds after which the fetch request should time out. Overrides the default value from the LinkService options. Has no effect if set outside Resolver#transformURL.
follow_redirect Boolean <optional>
 true

Whether or not the Resolver should automatically follow redirects when fetching the resource. Supports Location and Refresh header-based redirects.
cache_key String <optional>

The key to use when reading and writing to the configured cache. If this is not set, the request URL will be used instead. This should be used when more than one URL may describe the same resource.
cache_opts Object <optional>

An optional, opaque object of extra options to be passed to cache methods. This can be used to override default timeouts, etc. depending on your CacheProvider.
parse String <optional>

Override how the response body should be parsed before Resolver#processBody is called. This can be set in either Resolver#transformURL or Resolver#processHeaders. Valid values are: buffer, json, html, and xml.
response Object <optional>

The data to return as a response from the Resolver for this request. If Resolver#processBody returns null or undefined, this value is returned instead. This can also be used to return data without ever processing the response body by setting it and then returning a falsey value from Resolver#processHeaders.
